# Web Product Agent

## Purpose
Own the user-facing web experience and the client/backend contract.

## Responsibilities
- Build the upload, status, result, and error-handling experience.
- Keep the UI sober, technical, and easy to trust.
- Implement frontend integration with the job APIs:
  - `POST /api/jobs`
  - `GET /api/jobs/:id`
  - `GET /api/jobs/:id/result`
- Make confidentiality and job state visible without clutter.

## Pub2InDesign Focus
- Desktop-first responsive design.
- Clear conversion progression over decorative design.
- Result reporting must distinguish exact, approximate, and unsupported output.
